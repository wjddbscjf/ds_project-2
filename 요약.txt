직원정보 데이터
- 이름 (name) 부서코드(dept_no) 사번(ID) 연봉 (annual income) 존재 EmployeeData.h안에 private 멤버 변수로 선언되어 있음.
- LOAD 명령어로 employee.txt에 있는 정보들을 클래스에 저장, 탭 '\t'로 정보들이 구분되어 있음. 
 부서코드는 100~800까지 100단위로 존재함. 각 분류에 따라서 Selection Tree의 Leaf Node 구성.

B+ tree.
- LOAD 명령어가 실행되면 B+ 트리 데이터 구조에 이름을 기준으로 해서 정렬되어서 저장됨.
  B+ 트리에 저장되는 데이터들은 모두 EmployeeData로 선언되어있어야함.

Selection Tree
- 셀렉션 트리의 run의 개수는 부서코드의 개수이다. 셀렉션 트리의 노드의 내부 데이터는 EmployeeData로 선언되어있기에, 배열이 아닌 포인터 형태로 구현한다. 각 run들은 EmployeeHeap class로 구현되며, Heap이 정렬되면 셀렉션 트리도 재정렬한다.

EmployeeHeap
- 맥스힙 구조를 고수하며, 포인터 배열로 구현하며, 동적할당을 이용해야한다. ADD_ST명령어를 통해 비플트리에서 복사해온 데이터를 Heap으로 구축한다. Heap에 새로운 데이터 추가는 왼쪽 자식 노드부터 생성된다.
ADD_ST명령어를 사용할 때마다 힙은 재정렬된다. 
Heap이 재정렬될때 모든 부모 노드 연봉이 자식 노드의 연봉보다 크거나 같은 경우에 정렬을 완료한다. 
0번 노드는 사용하지않고, leftchild는 2n, rightchild는 2n+1이다. 


LOAD
- 이전 프로젝트의 LOAD와 비슷하나 '\t'를 구분자로 정보가 저장되어 있다.

ADD_BP
- 비플트리에 직접 사원을 추가하는 명령어로, 총 4개의 인자를 사용한다.
name/100(부서코드)/ 220001(사번)/9000(연봉) 
인자가 하나라도 존재하지 않는다면 에러코드를 출력하고 <- 해당 경우만 오류를 출력한다.
#주어진 부서코드 이외의 코드를 입력하는 경우는 존재하지 않는다.

SEARCH_BP
- 비플트리에 저장된 데이터를 검색하는 명령어로, 1개 또는 2개의 인자를 추가로 입력

인자가 한개인 경우는 해당 이름의 검색 결과를 출력한다. 

인자가 두개인 경우는 해당 범위 검색 결과를 출력한다.
- 예를들어 인자로 a와c가 입력되면 이름이 a로 시작하는 사원부터 c로 시작하는 사원을 모두 출력한다. a<= ? <=c 검색하	  는 인자를 입력하지 않은 경우, B+트리에 데이터가 없는 경우 검색하는 이름의 사원이 없는 경우에 에러 코드 출력

PRINT_BP
- 비플 트리에 있는 모든 정보 오름차순 기준으로 출력

ADD_ST
- 비플 트리에 있는 사원 데이터를 셀렉션 트리로 복사하는 명령어, 2개의 인자를 필요로 하며, 데이터를 불러오는 방식은 2가지

1. 부서코드를 이용해서 불러오는 방식
2. 사원 이름을 통해서 불러오는 방식

dept_no(부서코드)를 이용해서 불러오는 방식은 비플트리 내의 오름차순 맨 앞의 데이터부터 맨 끝의 데이터를 모두 확인해서 검색을 진행한다. 
name을 통해서 검색하는 경우는  SEARCH_BP 명령어를 활용하는 것처럼 IndexNode를 통해 검색 진행.
명령어를 실행할 때 비플트리에는 데이터가 존재해야하며, 각 부서코드에 알맞은 Heap이 삽입되고, 셀렉션 트리의 업데이트가 일어나야한다. 만약 검색한 데이터가 존재하지 않거나, 알 수 없는 인자를 받은 경우 에러코드 출력

PRINT_ST
- 셀렉션 트리에서 인자로 받은 코드에 해당하는 데이터를 출력하는 명령어로 1개의 인자를 입력받는다. 인자는 부서코드(dept_no)이고 출력은 사원 이름을 기준으로 오름차순으로 출력한다. 

1. 셀렉션 트리에 저장된 데이터가 없는 경우
2. 사원(부서 ?)코드에 대응되는 Heap 자료구조가 없는 경우
3.잘못된 부서코드를 입력하는 경우
위 세가지 경우에서만 에러코드를 출력한다. 

DELETE
- 셀렉션 트리에서 root node에 저장된 사원을 제거하는 명령어이다. 이떄 사원이 제거되어 Heap에 저장된 사원 연봉의 대소관계가 변경되면 Heap을 재정렬한다. 자료구조에 데이터가 존재하지 않을 시 에러 코드 출력.
